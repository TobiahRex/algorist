1Ô∏è‚É£ Linear Chain

Each state depends on previous state(s). The simplest DP shape.

dp[i] = f(dp[i-1]) or f(dp[i-1], dp[i-2])

Problems with this shape:

  • Fibonacci, Climbing Stairs
  • House Robber
  • Maximum Subarray (Kadane's)
  • Decode Ways

2Ô∏è‚É£ Decision Tree

Each node = choice point. Branches = options. Leaves = base cases.

dp[i] = max(take + dp[i+1], skip + dp[i+1])

Problems with this shape:

  • 0/1 Knapsack
  • Subset Sum, Partition Equal
  • Target Sum (+/-)
  • Coin Change (unbounded)

3Ô∏è‚É£ Grid / 2D Table

Two dimensions of state. Cell depends on neighbors (usually up/left).

dp[i][j] = f(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])

Problems with this shape:

  • Longest Common Subsequence
  • Edit Distance
  • Unique Paths in Grid
  • Pascal's Triangle!

4Ô∏è‚É£ Interval / Range

Subproblems defined by [start, end]. Expand outward or split in middle.

dp[i][j] = min/max over k: f(dp[i][k], dp[k][j])

Problems with this shape:

  • Longest Palindromic Substring
  • Matrix Chain Multiplication
  • Burst Balloons
  • Palindrome Partitioning

5Ô∏è‚É£ Subset / Bitmask

State = which elements included. 2‚Åø subsets form a lattice.

dp[mask] = f(dp[mask ^ (1 << i)]) for each bit i

Problems with this shape:

  • Traveling Salesman (TSP)
  • Assign Tasks to Workers
  • Counting Subsets with Property
  • Shortest Hamiltonian Path

üí° The Pascal's Triangle Insight

You discovered something profound: Pascal's Triangle is simultaneously all of these shapes:

‚Ä¢ Grid: Row i, position j ‚Üí depends on [i-1][j-1] and [i-1][j]

‚Ä¢ Decision Tree: Each path from top = sequence of left/right choices. Nodes that reconverge = overlapping subproblems!

‚Ä¢ Linear Chain: Each row can be computed from the previous row alone

The "hidden tree" you saw is the KEY insight: every DP problem has an implicit exponential tree, but overlapping subproblems let us collapse it into polynomial space.
